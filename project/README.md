## 2. Структура проекта

Проект организован в следующей структуре:

- `requirements.txt` – зависимости проекта (библиотеки Python, необходимые для запуска).
- `report.md` – отчёт по проекту (постановка задачи, данные, эксперименты, результаты).
- `self-checklist.md` – чеклист самопроверки проекта перед сдачей.
- `notebooks/` – экспериментальные ноутбуки:
  - EDA, прототипы моделей, отдельные эксперименты.
- `src/` – основной код проекта:
  - модули для работы с данными;
  - обучение моделей;
  - запуск сервиса/скриптов.
- `data/` – демонстрационные/учебные данные:
  - **только открытые, синтетические или обезличенные данные;**
  - при необходимости – небольшие выборки для быстрого запуска.
- `configs/` – конфигурационные файлы:
  - `.yaml`, `.json`, `.env.example` и т.п.
- `tests/` – тесты (юнит-тесты, простые проверки).
- `artifacts/` – сохранённые модели, отчёты, результаты экспериментов.

## 3. Требования и установка

### 3.1. Требования

- Python `>= 3.10` (уточните при необходимости).
- Установленные системные зависимости (если есть: например, `git`, `make` и т.п.).

### 3.2. Установка окружения

```bash

# Создать виртуальное окружение (опционально, но рекомендуется)

# Активировать окружение:

source .venv/bin/activate

## 4. Как запустить проект

### 4.1. Запуск обучения модели

```
Если по каким-либо причинам модель не сохранена и сервис не может ее загрузить: 

1. Открыть ноутбук notebooks/item_based_collaborative_filtering.ipynb
2. Запустить все ячейки последовательно
3. После обучения модель автоматически сохранится по пути: src/models/usercf_model.pkl
```

### 4.2. Запуск сервиса (API/веб-интерфейс)

```bash
cd project
source .venv/bin/activate 
uv run uvicorn src.service.main:app --reload --host 127.0.0.1 --port 8000
```

Или(альтернативный вариант):

```bash
cd project
docker build -t movie-recommender .
docker run -d -p 8000:8000 --name recommender movie-recommender
```

Опишите:

- сервис поднимается на порту 8000;
- какие эндпоинты есть:
`/health` - базовая проверка работоспособности сервиса. Показывает статус, сколько имеется в базе фильмов и пользователей.
`/recommend/{user_id}` - выдача рекомендаций по id конкретному пользователю


- как протестировать работоспособность (например, запрос через `curl` или браузер) Пример на эндпоинте `/health`: curl http://127.0.0.1:8000/health.
